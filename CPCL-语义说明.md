# CPCL 语义说明（数字参数版）

本文档专门解释本项目中各 CPCL 指令里“每个数字”的含义，统一按 dots（打印点）理解坐标与尺寸。

## 1. 通用约定

- 坐标原点为左上角 `(0,0)`。
- `x` 向右增大，`y` 向下增大。
- 未特别声明时，数字单位均为 dots。

## 2. 页面与尺寸指令

### `! 0 200 200 <height> <qty>`

| 位置 | 示例值 | 含义 |
|---|---:|---|
| 第1个数字 | `0` | 偏移（本项目不使用，通常固定写 0） |
| 第2个数字 | `200` | 水平分辨率（DPI） |
| 第3个数字 | `200` | 垂直分辨率（DPI） |
| 第4个数字 | `<height>` | 标签高度；项目解析时会用它作为画布高度 |
| 第5个数字 | `<qty>` | 打印份数；项目重建发送时固定为 `1` |

### `SIZE <width> <height>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 画布宽度 |
| 第2个数字 | 画布高度 |

### `PW <width>` / `PAGE-WIDTH <width>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 页面宽度 |

### `PH <height>` / `PAGE-HEIGHT <height>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 页面高度 |

## 3. 文本指令

### `TEXT <x> <y> <text...>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 文本起点 `x` |
| 第2个数字 | 文本起点 `y` |

### `TEXT <font> <size> <x> <y> <text...>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 字体编号（项目解析时仅做兼容，不参与预览渲染计算） |
| 第2个数字 | 字号/字模大小（项目解析时仅做兼容） |
| 第3个数字 | 文本起点 `x` |
| 第4个数字 | 文本起点 `y` |

### `TEXT90|TEXT180|TEXT270 <font> <size> <x> <y> <text...>`

| 指令 | 旋转角度 |
|---|---:|
| `TEXT90` / `T90` / `VT` | 90° |
| `TEXT180` / `T180` | 180° |
| `TEXT270` / `T270` | 270° |

数字位序与 `TEXT <font> <size> <x> <y> ...` 相同。

## 4. 线与框指令

### `LINE <x1> <y1> <x2> <y2> [thickness]`
### `INVERSE-LINE <x1> <y1> <x2> <y2> [thickness]`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 起点 `x1` |
| 第2个数字 | 起点 `y1` |
| 第3个数字 | 终点 `x2` |
| 第4个数字 | 终点 `y2` |
| 第5个数字（可选） | 线宽 `thickness`，默认 `1` |

### `BOX <x> <y> <width> <height> [thickness]`
### `INVERSE-BOX <x> <y> <width> <height> [thickness]`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 左上角 `x` |
| 第2个数字 | 左上角 `y` |
| 第3个数字 | 宽度 `width` |
| 第4个数字 | 高度 `height` |
| 第5个数字（可选） | 边框线宽 `thickness`，默认 `1` |

说明：项目内部语义是 `width/height`；发送重建时会转换成右下角坐标。

## 5. 一维码指令（Code128）

### `BARCODE128 <x> <y> <height> <data...>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 条码左上角 `x` |
| 第2个数字 | 条码左上角 `y` |
| 第3个数字 | 条码高度 `height` |

项目中此写法会使用默认窄条宽度 `ModuleWidth=2`。

### `BARCODE 128 <wide> <narrow> <height> <x> <y> <data...>`
### `VBARCODE 128 <wide> <narrow> <height> <x> <y> <data...>`

| 位置 | 含义 |
|---|---|
| 固定值 | `128` 表示 Code128 |
| 第1个数字 | `wide`（当前项目解析后不参与绘制） |
| 第2个数字 | `narrow`（项目作为模块宽度 `ModuleWidth` 使用） |
| 第3个数字 | 条码高度 `height` |
| 第4个数字 | `x` |
| 第5个数字 | `y` |

`VBARCODE` 表示竖向条码；`BARCODE` 为横向条码。

## 6. 二维码指令

### 单行模式：`QRCODE <x> <y> <moduleSize> <data...>`

| 位置 | 含义 |
|---|---|
| 第1个数字 | 左上角 `x` |
| 第2个数字 | 左上角 `y` |
| 第3个数字 | 模块尺寸 `moduleSize`（单个黑白格边长） |

### 块模式

```cpcl
QRCODE <x> <y> M 2 U <moduleSize>
MA,<data>
ENDQR
```

| 位置 | 含义 |
|---|---|
| 第1个数字 | 左上角 `x` |
| 第2个数字 | 左上角 `y` |
| 固定值 `2` | 容错级别参数（项目按原文透传语义处理） |
| `U` 后数字 | 模块尺寸 `moduleSize` |

兼容行为：若该行未取到有效模块尺寸，项目默认使用 `6`。

## 7. 控制类指令（兼容但不参与渲染）

`FORM`、`PRINT`、`END`、`CLS`、`SETMAG`、`SETLP`、`LEFT`、`RIGHT`、`CENTER`、`BARCODE-TEXT`、`COUNTRY`、`CODEPAGE`

这些指令中的数字参数不会进入预览绘制，但会影响实机打印行为（尤其是编码与打印流程控制）。

## 8. 发送重建时的数字调整规则

`cpcl-bluetooth` 在发送前会重建 CPCL，常见数字行为如下：

- 头部固定重建为 `! 0 200 200 <scaledHeight> 1`。
- `PW/PH` 使用缩放后的宽高。
- 文本重建为 `TEXT* 24 <fontSize> <x> <y> <text>`，其中 `fontSize` 会按比例并设置下限。
- 条码重建为 `BARCODE/VBARCODE 128 <module> <module> <height> <x> <y> <data>`，即 `wide` 与 `narrow` 会被统一。
- 二维码重建为 `B QR <x> <y> M 2 U <moduleSize>`，并带最小尺寸限制。

如果你要“严格保持原始数字不变”，可在 `app_config.json` 设置：

```json
"print": {
  "skip_rebuild": true
}
```

## 9. 为什么会出现“数字偏差”

常见原因如下：

- **设备兼容性保护**：不同固件对 `TEXT/BARCODE/QR` 的参数解释不同，发送前重建会做统一化处理。
- **缩放收敛**：当内容宽度超过目标纸宽时，程序会按比例缩放坐标与尺寸，导致数值变化。
- **最小值保护**：线宽、条码模块宽、二维码模块宽会设置最小阈值，避免缩放后无法打印或难以识读。
- **结构语义转换**：`BOX` 在内部使用 `x/y/width/height` 语义，发送时会转换成 `x1/y1/x2/y2`。

## 10. 偏差排查步骤

1. 先开启导出重建文件：

```json
"print": {
  "export_rebuild_cpcl": true,
  "rebuild_cpcl_path": "./output/rebuild_cpcl.txt"
}
```

2. 打印后对比原始输入与导出文件，重点关注：`! 0 200 200 ...`、`PW/PH`、`TEXT*`、`BARCODE/VBARCODE`、`B QR ... U`。
3. 若确认偏差由重建引起，临时设置 `skip_rebuild=true` 复测。
4. 若关闭重建仍有异常，再检查编码、蓝牙特征 UUID、设备固件与纸张规格。
